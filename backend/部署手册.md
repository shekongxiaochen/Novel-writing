# 部署手册（Docker 生产部署）

本手册带你把后端部署到云服务器，并让打包的客户端能连上。
全程在**服务器**上操作（除最后重新打包客户端在你本机）。

前置：你已有域名；服务器待购（Linux，建议 2核4G 起，Ubuntu 22.04）。

---

## 第 0 步：买服务器 + 域名解析

1. 买一台云服务器（阿里云/腾讯云/Vultr 等），系统选 **Ubuntu 22.04**，记下公网 IP。
2. 在域名服务商把一条记录解析到服务器 IP：
   - 类型 `A`，主机记录 `api`，记录值 = 服务器公网 IP
   - 这样 `api.你的域名.com` 就指向服务器了
3. 在云服务商控制台的**安全组/防火墙**放行端口：`22`(SSH)、`80`(HTTP)、`443`(HTTPS)。
   - **不要**放行 3306/6379/8080（数据库和后端只走内网，公网不可达）。

---

## 第 1 步：登录服务器，装 Docker

```bash
ssh root@你的服务器IP

# 安装 Docker + compose 插件
curl -fsSL https://get.docker.com | sh
docker compose version   # 验证，应打印版本号
```

---

## 第 2 步：上传后端代码到服务器

在**你本机**，把 `backend/` 目录传到服务器（任选其一）：

```bash
# 方式A：scp（在本机 d:/Novel writing 目录下运行）
scp -r backend root@你的服务器IP:/opt/novel-backend

# 方式B：git clone（若代码在私有仓库）
ssh root@你的服务器IP
git clone <你的仓库地址> /opt/novel-backend
```

之后所有命令都在服务器的 `/opt/novel-backend` 里运行：

```bash
cd /opt/novel-backend
```

---

## 第 3 步：配置生产环境变量

```bash
cp .env.prod.example .env.prod
nano .env.prod        # 编辑
```

要改的：
- `CORS_ORIGINS` → 改成你的域名 `https://api.你的域名.com`
  （纯 Tauri 客户端其实不经过 CORS，但若以后有网页版就需要，先填上无妨）
- 其余密钥模板里已是随机强值，可直接用。

⚠️ 记下 `.env.prod` 里的 `ADMIN_PASSWORD`，登录管理后台要用。

---

## 第 4 步：先签 HTTPS 证书

把 nginx 配置里的域名占位改成你的真实域名：

```bash
sed -i 's/your-domain.com/api.你的域名.com/g' nginx.prod.conf
```

用 certbot 申请免费证书（standalone 模式，需 80 端口临时空闲）：

```bash
mkdir -p ssl certbot/www
docker run --rm -p 80:80 \
  -v $(pwd)/ssl:/etc/letsencrypt/live-out \
  certbot/certbot certonly --standalone \
  -d api.你的域名.com --register-unsafely-without-email --agree-tos \
  --deploy-hook "cp /etc/letsencrypt/live/api.你的域名.com/*.pem /etc/letsencrypt/live-out/"
```

成功后把证书放到 nginx 期望的位置：

```bash
ls ssl/    # 应能看到 fullchain.pem 和 privkey.pem
```

> 若上面 deploy-hook 没生成文件，手动复制：
> `cp /etc/letsencrypt/live/api.你的域名.com/fullchain.pem ssl/`
> `cp /etc/letsencrypt/live/api.你的域名.com/privkey.pem ssl/`

---

## 第 5 步：启动全部服务

```bash
docker compose -f docker-compose.prod.yml --env-file .env.prod up -d --build
```

首次会编译 Rust（几分钟）。完成后检查：

```bash
docker compose -f docker-compose.prod.yml ps     # 各服务应为 healthy/running
curl -k https://api.你的域名.com/health          # 应返回健康响应
```

数据库表会在 MySQL 首次启动时由 `migrations/*.sql` 自动建好。

---

## 第 6 步：固化管理员密码

首次启动用 `ADMIN_SYNC_PASSWORD=true` 把密码写进了数据库。现在改回 false：

```bash
sed -i 's/ADMIN_SYNC_PASSWORD=true/ADMIN_SYNC_PASSWORD=false/' .env.prod
docker compose -f docker-compose.prod.yml --env-file .env.prod up -d
```

管理后台地址：`https://api.你的域名.com/admin/`
（生产环境建议进一步限制管理后台访问来源，例如只允许你的 IP，后续可加。）

---

## 第 7 步：重新打包客户端（指向生产后端）

回到**你本机**，告诉我你的域名，我帮你改 `frontend/.env.local` 的
`VITE_API_BASE_URL=https://api.你的域名.com`，然后重新打包：

```bash
cd frontend
npm run tauri:build
```

产物在 `frontend/src-tauri/target/release/bundle/`，这个包发给用户就能连上线上后端了。

---

## 日常运维速查

```bash
# 看日志
docker compose -f docker-compose.prod.yml logs -f backend

# 重启后端
docker compose -f docker-compose.prod.yml restart backend

# 更新代码后重新部署
git pull   # 或重新 scp
docker compose -f docker-compose.prod.yml --env-file .env.prod up -d --build

# 备份数据库
docker exec novel-mysql mysqldump -uroot -p"$MYSQL_ROOT_PASSWORD" novel_db > backup_$(date +%F).sql
```

### 证书续期（Let's Encrypt 90 天到期）
```bash
docker run --rm -p 80:80 certbot/certbot renew
# 续期后重新复制 pem 到 ssl/ 并 restart nginx
```
