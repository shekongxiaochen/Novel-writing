# 启动说明

> 本文档面向从源码运行 / 自建后端的开发者。仅使用桌面安装包的用户无需关注。

后端仅保留 **Rust**（目录 `backend/`），默认端口 **8080**。需本机 **MySQL** 与 **Redis**，以及 Rust 稳定版工具链。

## 前端

```bash
cd frontend
npm install

# Web 调试（浏览器，端口 5174）
npm run dev

# 桌面应用开发模式（Tauri 窗口）
npm run tauri:dev
```

- Web 调试默认地址：`http://127.0.0.1:5174`
- API 地址：在 `frontend/.env.local` 中配置 `VITE_API_BASE_URL=http://127.0.0.1:8080`

不启动后端也能完整本地写作，后端仅用于账号与云同步。

## 后端（Rust，可选）

```bash
cd backend
cp .env.example .env   # 首次：复制并按需修改配置
cargo run
```

- 健康检查：`http://127.0.0.1:8080/health`
- 数据库检查：`http://127.0.0.1:8080/health/db`
- **管理后台**：`http://127.0.0.1:8080/admin/`（详见 [docs/admin-console.md](docs/admin-console.md)）

在 `backend/.env` 中启用管理后台：

```env
ADMIN_ENABLED=true
ADMIN_USERNAME=admin
ADMIN_PASSWORD=请设置为强密码
```

> 安全提示：务必为 `ADMIN_PASSWORD` 设置强密码，切勿使用默认值或将真实密码提交到仓库。

## Redis

按你本机的安装方式启动 Redis 服务（默认端口 `6379`），后端启动前需确保 Redis 可用。

## 当前约定

- 前端端口：`5174`
- 后端端口：`8080`（仅 Rust）
- 不再使用 Python / FastAPI 后端
