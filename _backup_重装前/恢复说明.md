# 重装系统后的恢复说明

这个目录是「重装系统前的完整备份」，包含正常不进 git 的敏感文件和数据。
**仓库是私有的，所以这些可以安全地放在这里。重装后照下面操作即可全部找回。**

## 文件清单
- `backend.env.bak` —— 后端环境变量（DeepSeek 密钥、数据库密码、JWT、admin 密码等）
- `novel_db.sql` —— 本地 MySQL 数据库 novel_db 的完整导出（用户、小说、钱包、设置等）

## 恢复步骤（重装系统、重新 clone 仓库后）

1. 还原后端 .env：
   ```bash
   cp _backup_重装前/backend.env.bak backend/.env
   ```

2. 还原数据库（先装好 MySQL，建库后导入）：
   ```bash
   mysql -h127.0.0.1 -P3306 -uroot -p < _backup_重装前/novel_db.sql
   ```
   （novel_db.sql 里已含 CREATE DATABASE，直接导入即可）

3. 重建依赖与编译产物（这些没备份，重新生成即可）：
   ```bash
   cd backend && cargo build
   cd ../frontend && npm install && npm run build
   ```

## 提醒
- 这些密钥是本地开发用的。生产服务器的密钥另在服务器上，与此无关。
- 恢复完成、确认一切正常后，可以删掉本目录并从 git 移除，避免长期把密钥留在仓库历史里。
